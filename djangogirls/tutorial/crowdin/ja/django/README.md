# ジャンゴってなに？

ジャンゴはPythonで書かれた、フリーでオープンソースのwebアプリケーションフレームワークです。 Web フレームワークは、素早く、簡単にウェブサイトを開発するのに役立つコンポーネントの一式が含まれています。

Web サイトを構築するとき常に同じような一連のコンポーネントを必要があります: ユーザー認証 (登録、サインイン、サインアウト) を処理する方法、あなたのウェブサイト、フォーム、ファイルなどをアップロードする方法の管理パネルです。

幸いにも新しくサイトを構築する際に、web 開発者が直面する同じような問題に対してずっと前に気づいた彼らは協力して既製品のフレームワークを作成しました。 (ジャンゴはそれらの 1 つです。)

フレームワークはサイトを一から作ることから救うために存在し、新しいサイトを構築する場合、オーバーヘッドを減らせます。

## なぜフレームワークを必要とするか。

ジャンゴが実際にどういうものか理解するために、サーバーで詳しく見る必要があります。まずは、サーバーが web ページを提供することについて知っている必要があります。

やってくるメールを監視するメール受信箱を想像してみてください。 これはWEBサーバによってです。 Webサーバはメールを受け取り、レスポンスをWEBページに返します。 しかし、何かをレスポンスを返す場合は、いくつかのコンテンツを持っている必要があります。 Djangoは、コンテンツを作成することができます。

## 誰かがあなたのWEB サイトにリクエストを要求したときどうなりますか。

Webサーバーへリクエストが来ると、リクエストされたものを理解してジャンゴに渡します。 それは web ページのアドレスを最初に渡されるし、何をするかを理解します。 これはDjangoの **urlresolver** ( *urlresolver* 理にかなってので、ウェブサイトのアドレスはURL - ユニフォーム リソース ロケーター - 呼ばれます) によって行われます。 非常にわかりづらいのですが、 それはパターンのリストを受け取り、URL マッチをします。 Djangoは、上から下に、パターンをチェックし、関連する関数へリクエストを渡します。(*View* と呼びます。)).

配達人と手紙を想像してみてください。 彼女は通りを歩き、各家の番号をチェックします。 手紙とマッチすると、手紙を配達します。 これがurlresolverがどのように動いているか、です。

ビュー関数において、これら全てのことが行われます：情報を検索するためのデータベースを見ることができます。 ユーザはデータにおいて何か変更するために尋ねるだろうか？ 手紙は言います。「自分の機能の説明を変更してください。」ビューはそれが許可されているかどうか確認し、機能の説明を更新し、「完了」のメッセージを送ります。 ビューはレスポンスを返し、Djangoはユーザのwebブラウザに返します。
